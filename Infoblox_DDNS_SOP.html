Infoblox DDNS Settings - SOP

1. DDNS Updates

Checkbox: Enable DDNS Updates

Purpose: Controls whether DHCP leases trigger updates to the DNS server.

  • If enabled, when DHCP assigns an IP address to a host, Infoblox automatically registers (or updates) the hostname and IP in DNS.
  • If disabled, no dynamic DNS updates occur.

2. DDNS Domain Name

Field: DDNS Domain Name

Purpose: Defines the domain name into which host records are created when DDNS is enabled.

Example: If the domain is set to lab.local, a host with hostname PC1 will be registered as PC1.lab.local.

3. DDNS Update TTL

Field: Time-to-Live value for DDNS entries

Purpose: Specifies how long the dynamically created DNS records remain cached in DNS.

Example: If set to 3600 seconds (1 hour), DNS records will expire and need to be refreshed after 1 hour.

4. Generate Hostname

Checkbox: Generate Hostname if not Sent by Client

Purpose: If a DHCP client doesn’t provide a hostname, Infoblox can auto-generate one.

Example: A client without a hostname may be assigned host-192-168-10-15.lab.local.

5. Fixed Address Updates

Checkbox: Update Fixed Addresses

Purpose: Controls whether Infoblox should also perform DNS updates for fixed-address clients (devices with a reserved/static IP from DHCP).

Useful if you want DNS records to stay in sync with devices that always get the same IP.

⚠️ Important Note: Even if you configure domain name, TTL, etc., DDNS updates won’t happen unless you explicitly enable “DDNS Updates” at this network level (or inherit it from Grid/global settings).

6. Option 81 Support

Checkbox: Enable Option 81 Support

Purpose: DHCP Option 81 = Client FQDN (Fully Qualified Domain Name) option. It allows a DHCP client to tell the server its hostname and whether the server or the client should update DNS.

  • DHCP server always updates DNS: The DHCP server updates the DNS records for the client regardless of what the client requests.
  • DHCP server updates DNS if requested by client: The DHCP server only updates DNS if the client specifically asks for it in the Option 81 request.

7. Lease Renewal Update

Checkbox: Update DNS on DHCP Lease Renewal

Purpose: Controls whether the DNS records are updated when a client renews its DHCP lease.

  • If enabled, each time the client renews its lease, Infoblox refreshes the DNS record with the same hostname and IP.
  • If disabled, DNS is only updated on the initial lease assignment (not on renewals).

Tip: Enabling helps prevent stale DNS entries; disabling reduces DNS traffic.
