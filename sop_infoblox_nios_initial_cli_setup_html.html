SOP: Infoblox NIOS – Initial CLI Setup (Standalone Node)

Author: Suraj · Role: Network Lead · Last updated: 18 Aug 2025 · Scope: Lab/POC
Product: NIOS 9.0.3 Appliance: IB‑V815 (selected) Mode: Standalone (no Grid join)

Purpose

Provision a fresh Infoblox NIOS VM from the CLI, set IPv4 networking, apply temporary licenses for DNS, DHCP & Grid, and verify basic reachability.

Prerequisites

Console Access

  • Hypervisor/Cloud serial console or VMRC/VNC.
  • Default admin credentials available.

  4. Access the web UI.

    Browse to https://192.168.100.200. Accept the certificate warning. Log in with admin and your password.

Validation Checklist

  • CLI shows LAN IPv4 as 192.168.100.200/24.
  • Web UI loads over HTTPS and reflects temporary licenses.
  • (If required) Ping/ARP to default gateway succeeds.

Troubleshooting Quick Hits

  • No gateway reachability: Verify switchport access VLAN or trunk native VLAN; ensure SVI 192.168.100.254 exists and is up; check ACLs.
  • License not visible in UI: Restarted UI? Re‑login. From CLI, run show license.
  • Wrong IP entered: Re‑run set network; confirm before reboot.
